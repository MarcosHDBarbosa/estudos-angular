Dificuldades

Aqui vai um resumo das principais dificuldades comuns para iniciantes

  1. Misturar responsabilidades (HTML, CSS e JavaScript)
    • Iniciantes tendem a usar HTML para aparência (“deixar bonito”), quando na verdade ele serve para estrutura e significado.
    • Dificuldade em separar:
      • HTML → estrutura
      • CSS → visual
      • JavaScript → comportamento
  2. Falta de entendimento de semântica
    • Uso excessivo de <div> para tudo.
    • Dificuldade em escolher elementos corretos como <header>, <main>, <section>, etc.
    • Não entender que HTML comunica significado, não só estrutura.
  3. Estrutura e hierarquia do documento
    • Problemas para entender:
      • Estrutura básica (<!DOCTYPE>, <html>, <head>, <body>)
      • Organização do conteúdo
    • Dificuldade em montar páginas completas corretamente.
  4. Erros de aninhamento (muito comum)
    • Fechar tags fora de ordem:
      •               
                  <p><strong>Texto</p></strong>
    • Não entender a lógica de hierarquia (pai/filho).
  5. Uso incorreto de atributos
    • Esquecer aspas:
      •               
                  <img src=logo.png>
    • Não entender função de atributos como href, src, alt, id, class.
  6. Problemas com imagens
    • Não usar atributo alt ou usar de forma inútil (alt="imagem").
    • Caminhos errados → imagem “quebrada”.
  7. Hierarquia incorreta de títulos
    • Pular níveis (ex: <h1> → <h4>).
    • Escolher título pelo tamanho visual, não pela estrutura.
  8. Links pouco descritivos
    • Uso de “clique aqui”.
    • Falta de clareza sobre o destino do link.
  9. Dificuldade com caminhos de arquivos
    • Erros em caminhos relativos:
      • imagens não carregam
      • links quebrados
  10. Falta de validação e depuração
    • Não usar validadores HTML.
    • Não identificar erros como:
      • tags não fechadas
      • atributos mal escritos
  11. Ignorar acessibilidade
    • Não usar elementos semânticos.
    • Não associar <label> a inputs.
    • Não entender importância de leitores de tela.
  12. Pensar em aparência antes do significado
    • Exemplo clássico:
      • usar <div> estilizada em vez de <h1></h1>
    • Falta da mentalidade correta:
      • Deve pensar “o que isso significa?” em vez de “como isso parece?”

No fundo, quase todos os problemas vêm de um ponto central, não entender que HTML é sobre significado e estrutura, não aparência
